# Contributing

Document status: **Current**

Before adding or revising documentation, read the
[development system](../development-system.md) and run
`bash scripts/check-documentation-system.sh` from the repository root.

BridgeVM follows the rulebook from `PLAN.md`:

- Keep Fast Mode narrow.
- Put advanced hardware control in Compatibility Mode.
- Treat guest tools as a first-class product surface.
- Make everything scriptable through CLI/API.
- Avoid promises about Windows graphics acceleration until the engineering exists.
